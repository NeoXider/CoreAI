# MultiAgent Workflow Plan

## Цель
Реализовать multi-agent orchestration: Creator может управлять другими агентами (Programmer, CoreMechanic, Analyzer и др.) через tool calls.

## Важное уточнение
**Tool calling уже реализован через MEAI** (MemoryTool, LuaTool, InventoryTool). AgentTool должен использовать тот же паттерн.

```csharp
    public AIFunction CreateAIFunction() => AIFunctionFactory.Create(
```

**Tool Schema (JSON):**
```json
{
  "type": "object",
  "properties": {
    "action": {
      "type": "string",
      "enum": ["call", "update_prompt", "add_tool", "list_agents"],
      "description": "Action to perform"
    },
    "role": {
      "type": "string", 
      "description": "Target agent role (e.g., 'Programmer', 'CoreMechanicAI')"
    },
    "hint": {
      "type": "string",
      "description": "Task hint for 'call' action"
    },
    "prompt": {
      "type": "string",
      "description": "New system prompt for 'update_prompt' action"
    },
    "timeout": {
      "type": "integer",
      "description": "Timeout in seconds for 'call' action (default 60)"
    }
  },
  "required": ["action"]
```

**Actions:**
- `call` — запускает агента, возвращает результат
- `update_prompt` — меняет system prompt агента
- `add_tool` — добавляет инструмент агенту
- `list_agents` — возвращает список доступных агентов

### 3. AgentOrchestrator — выполнение вызова агента
**Файл:** `Assets/CoreAI/Runtime/Core/Features/Orchestration/AgentOrchestrator.cs` (NEW)

```csharp
public interface IAgentOrchestrator {
    Task<AgentCallResult> CallAgentAsync(string roleId, string hint, int timeoutSeconds = 60, CancellationToken ct = default);
    Task<AgentCallResult> UpdateAgentPromptAsync(string roleId, string newPrompt, CancellationToken ct = default);
    IAgentRegistry GetRegistry();
}

public sealed class AgentOrchestrator : IAgentOrchestrator {
    private readonly IAgentRegistry _registry;
    private readonly ILlmClient _llmClient;
    private readonly IAiGameCommandSink _commandSink;
    private readonly ISessionTelemetryProvider _telemetry;
    private readonly AiPromptComposer _promptComposer;
    
    public async Task<AgentCallResult> CallAgentAsync(string roleId, string hint, int timeoutSeconds, CancellationToken ct) {
        // 1. Проверить что агент существует
        if (!_registry.HasAgent(roleId)) {
            return new AgentCallResult { Ok = false, Error = $"Agent '{roleId}' not found" };
        }
        
        // 2. Создать AiTaskRequest для роли
        var request = new AiTaskRequest {
            RoleId = roleId,
            Hint = hint ?? "",
            TraceId = Guid.NewGuid().ToString("N")
        };
        
        // 3. Вызвать LLM напрямую (минуя полный orchestrator для скорости)
        // Использует AgentRegistry.Get(roleId) для получения system prompt
        var systemPrompt = _registry.Get(roleId).SystemPrompt;
        var result = await _llmClient.CompleteAsync(new LlmCompletionRequest {
            AgentRoleId = roleId,
            SystemPrompt = systemPrompt,
            UserPayload = hint,
            TraceId = request.TraceId
        }, ct);
        
        return new AgentCallResult {
            Ok = result.Ok,
            Content = result.Content,
            Error = result.Error,
            TraceId = request.TraceId
        };
    }
}
```

## Пример работы

```
User: "Create a wave of enemies"
  → Creator LLM вызывает agent tool
  → AgentTool: {action: "call", role: "CoreMechanicAI", hint: "Create wave parameters"}
  → AgentOrchestrator.CallAgentAsync("CoreMechanicAI", "Create wave parameters")
  → CoreMechanicAI LLM возвращает JSON с wave config
  → Creator LLM видит результат
  → Creator вызывает: {action: "call", role: "Programmer", hint: "Generate enemy_wave.lua"}
  → Programmer LLM генерирует Lua код
  → Результат возвращается
```

## Вопросы

1. **Таймаут** — 60 секунд по умолчанию достаточно?
   ✅ **Подтверждено** — 60 секунд

2. **Ошибки** — если агент не найден, возвращать ошибку?
   ✅ **Подтверждено** — возвращать ошибку в JSON, агент может исправить сам или проигнорировать

3. **Параллельность** — нужно ли поддерживать параллельный вызов?
   ✅ **Подтверждено** — да, но по умолчанию 1, добавить в CoreAISettings
