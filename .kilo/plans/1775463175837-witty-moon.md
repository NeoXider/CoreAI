# MultiAgent Workflow Plan

## Цель
Реализовать multi-agent orchestration: Creator может управлять другими агентами (Programmer, CoreMechanic, Analyzer и др.) через tool calls.

## Интеграция с Creator

### System Prompt для Creator
Обновить промпт Creator, чтобы он знал о AgentTool:

```
## Available Tools
You can call other AI agents using the `agent` tool:
- call: Run another agent with a task
- update_prompt: Change another agent's system prompt
- add_tool: Add a tool to another agent
- list_agents: Get list of available agents

Example: {"name": "agent", "arguments": {"action": "call", "role": "Programmer", "hint": "Create a sword"}}
```

## Пример использования

```csharp
// Регистрация агентов при старте игры
var programmerConfig = new AgentBuilder("Programmer")
    .WithSystemPrompt("You are a Lua programmer...")
    .WithTool(new LuaLlmTool())
    .WithMode(AgentMode.ToolsOnly)
    .Build();

var coreMechanicConfig = new AgentBuilder("CoreMechanicAI")
    .WithSystemPrompt("You calculate game mechanics...")
    .WithMemory()
    .Build();

// Регистрация в AgentRegistry
registry.Register(programmerConfig);
registry.Register(coreMechanicConfig);

// Добавление AgentTool для Creator
policy.SetToolsForRole("Creator", new[] { new AgentTool() });

// Вызов агента из игры или из LLM
var result = await agentOrchestrator.CallAgentAsync("Programmer", "Create a healing potion");
```

## Приоритеты реализации

1. **AgentRegistry** — базовая структура для хранения агентов
2. **AgentResult** — простой data class
3. **AgentOrchestrator** — логика вызова агента через AiTaskRequest
4. **AgentTool** — MEAI tool с функциями call/update_prompt/add_tool/list_agents
5. **Интеграция** — подключение к AiOrchestrator и обновление промпта Creator

## Вопросы к UX

1. **Синхронный или асинхронный вызов агентов?** — предлагаю async (текущая архитектура already async)
2. **Таймаут для вызова агента?** — добавить параметр в AgentTool, по умолчанию 60с
3. **Что делать если агент не найден?** — возвращать ошибку через AgentResult
